Chord System Properties

The following properties are recognized by Chord. The separator for list-valued properties can be either a blank space, a comma, a colon, or a semi-colon.

Property name

Type Description Default value (if any)
Chord resource properties
chord.main.dir location main directory in Chord's installation.
chord.lib.dir location Directory containing libraries needed by Chord. ${chord.main.dir}/lib/
chord.main.class.path path Classpath of Chord. It includes the path specified by property chord.java.analysis.path to allow running user-defined program analyses. See ${chord.main.dir}/build.xml
chord.bddbddb.class.path path Classpath of bddbddb. See ${chord.main.dir}/build.xml
chord.instr.agent.file location Shared library implementing Chord instrumentation agent. ${chord.main.dir}/lib/libchord_instr_agent.so (on Linux; .dll on Windows and .dylib on MacOS)
chord.javadoc.url string URL of the Javadoc location of program analyses. It is used when publishing targets (i.e. when property chord.publish.targets is set to true). http://chord.stanford.edu/javadoc/
Chord boot properties
chord.work.dir location Working directory during Chord's execution. current working directory
chord.props.file location Properties file loaded by ${chord.main.dir}/build.xml. Any of the below properties, as well as other user-defined properties to be passed to Chord (e.g. for user-defined analyses), may be set in this file. Each relative (instead of absolute) path element in the value of any property named chord.<...>.path set in this file is converted to an absolute path element with respect to the directory containing this file. ${chord.work.dir}/chord.properties
chord.max.heap string Maximum memory size of JVM running Chord. 1024m
chord.max.stack string Maximum thread stack size of JVM running Chord. 32m
chord.jvmargs string Arguments passed to JVM running Chord. "-showversion -ea -Xmx${chord.max.heap} -Xss${chord.max.stack}"
chord.bddbddb.max.heap string Maximum memory size of JVM running bddbddb. 1024m
Chord task properties
chord.build.scope bool Compute program scope (i.e. reachable classes and methods) using algorithm specified by properties chord.scope.kind and chord.reuse.scope. Program scope is computed regardless of the value of this property if another task (e.g. a program analysis to be run) needs it. false
chord.run.analyses string list List of names of program analyses to be run in order. ""
chord.print.rels string list List of names of program relations whose contents must be printed to files ${chord.out.dir}/<...>.txt where <...> denotes the relation name. ""
chord.publish.targets bool Publish all targets defined by program analyses in paths specified by properties chord.java.analysis.path and chord.dlog.analysis.path. false
Basic program properties
chord.main.class class Fully-qualified name of main class of given program (e.g. com.example.Main).
chord.class.path path Classpath of given program. ${sun.boot.class.path} is implicitly appended to it. ${sun.boot.class.path}
chord.src.path path Java source path of given program. ""
chord.run.ids string list List of IDs to identify runs of given program. 0
chord.args.XXX string Command-line argument string to be used for given program in run having ID XXX. ""
chord.runtime.jvmargs string Arguments to JVM when running given program. It is used when the program is executed to compute its scope (i.e. when property chord.scope.kind is set to dynamic) as well as when the instrumented program is executed for dynamic analyses. "-ea -Xmx1024m"
Program scope properties
chord.scope.kind [rta|dynamic] Algorithm to compute program scope (i.e. reachable classes and methods). Current choices include rta (static Rapid Type Analysis) and dynamic (dynamic analysis). rta
chord.reuse.scope bool Treat program scope as the classes and methods listed in files specified by properties chord.classes.file and chord.methods.file, respectively. Property chord.scope.kind is ignored if this property is set to true and the two files exist. false
chord.check.exclude string list List of prefixes of names of classes and packages to be excluded by program analyses. Interpretation of this property is analysis-specific. "java.,sun.,com.sun.,com.ibm.,org.apache.harmony.,joeq.,jwutil."
chord.instr.exclude string list List of prefixes of names of classes and packages to be excluded from being instrumented. Classes in package java.lang.ref are not instrumented regardless of the value of this property. "java.,sun.,com.sun.,com.ibm.,org.apache.harmony.,joeq.,jwutil."
Program analysis properties
chord.java.analysis.path path Classpath containing program analyses written in Java (i.e. @Chord-annotated classes). ${chord.main.dir}/classes/
chord.dlog.analysis.path path Path of directories containing program analyses written in Datalog (i.e. *.datalog and *.dlog files). ${chord.main.dir}/src/dlog/
chord.reuse.rels bool Construct program relations from BDDs stored on disk (from a previous run of Chord) whenever possible instead of re-computing them. false
chord.publish.results bool Publish the results of program analyses in HTML. Interpretation of this property is analysis-specific. true
Program transformation properties
chord.ssa bool Do SSA transformation for all methods deemed reachable by the algorithm used to compute program scope. true
Chord debug properties
chord.bddbddb.noisy bool Make bddbddb output debug info when solving Datalog programs. false
chord.save.maps bool Write to file ${chord.bddbddb.work.dir}/<...>.map when saving program domain named <...> (useful for debugging Datalog programs using debug target of ${chord.main.dir}/build.xml). true
Chord instrumentation properties
chord.trace.pipe bool Implement the dynamic trace file as a POSIX pipe instead of a regular file. true
chord.trace.block.size int Number of bytes to read/write in a single operation from/to the dynamic trace file. 4096
chord.runtime.class class Subclass of chord.project.Runtime used by dynamic program analyses to handle events generated during an instrumented program's execution. chord.project.BufferedRuntime
Chord output properties
chord.out.dir location Directory to which Chord dumps all files. ${chord.work.dir}/chord_output/
chord.out.file location File to redirect standard output stream during Chord's execution. ${chord.out.dir}/log.txt
chord.err.file location File to redirect standard error stream during Chord's execution. ${chord.out.dir}/log.txt
chord.classes.file location File from/to which list of classes deemed reachable is read/written. ${chord.out.dir}/classes.txt
chord.methods.file location File from/to which list of methods deemed reachable is read/written. ${chord.out.dir}/methods.txt
chord.bddbddb.work.dir location Directory used by bddbddb as its input/output directory. ${chord.out.dir}/bddbddb/
chord.boot.classes.dir location Directory containing instrumented JDK classes used by given program. ${chord.out.dir}/boot_classes/
chord.user.classes.dir location Directory containing instrumented non-JDK classes of given program. ${chord.out.dir}/user_classes/
chord.instr.scheme.file location File specifying kind and format of events in dynamic trace file. ${chord.out.dir}/scheme.ser
chord.crude.trace.file location Crude dynamic trace file. ${chord.out.dir}/crude_trace.txt
chord.final.trace.file location Final dynamic trace file. ${chord.out.dir}/final_trace.txt
